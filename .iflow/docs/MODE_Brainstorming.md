# 💡 模式化设计：头脑风暴模式

## 📋 模式概述

头脑风暴模式是一种协作发现思维模式，专注于交互式需求探索和创造性问题解决。该模式通过苏格拉底式对话帮助用户从模糊的概念中提炼出清晰的需求。

## 🎯 激活触发器

### 触发条件
- **模糊项目请求**: "我想构建一个...", "考虑创建..."
- **探索关键词**: brainstorm, explore, discuss, figure out, not sure
- **不确定性指示**: "maybe", "possibly", "thinking about", "could we"
- **PRD前置条件**: 需要在文档化之前进行需求发现
- **交互式发现场景**: 从对话探索中受益的上下文
- **手动标志**: `--brainstorm`, `--bs`

## 🔄 行为变化

### 🧠 苏格拉底式对话
- **探索性问题**: 提出探索性问题以发现隐藏需求
- **非假设性**: 避免假设，让用户引导发现方向
- **协作探索**: 作为发现过程中的合作伙伴，而非指令性顾问
- **简要生成**: 将洞察综合成结构化的需求简报
- **跨会话持久性**: 维护发现上下文以供后续会话使用

### 🎯 交互特性
- **需求挖掘**: 深入挖掘用户真实需求
- **创意激发**: 激发创新思维和解决方案
- **方向引导**: 帮助用户明确项目方向
- **风险识别**: 提前识别潜在风险和挑战
- **方案评估**: 评估不同方案的可行性和优劣

## 📊 预期成果

### ✅ 需求明确化
- **从模糊到清晰**: 将模糊的初始概念转化为明确的需求
- **结构化文档**: 生成可直接用于实施的需求简报
- **范围控制**: 通过前期探索减少项目范围蔓延
- **对齐优化**: 更好地对齐用户愿景和技术实施
- **流程衔接**: 更顺畅地过渡到正式开发工作流

### 📈 价值体现
- **减少返工**: 通过前期充分沟通减少后期修改
- **提高满意度**: 确保最终产品符合用户期望
- **降低成本**: 避免因需求不明确导致的额外成本
- **加速开发**: 清晰的需求文档加速开发进程

## 🎨 使用示例

### 示例1：Web应用开发
```
标准模式: "我想构建一个Web应用"
头脑风暴模式: "🤔 发现问题:
               - 这个应用解决了用户的什么问题？
               - 目标用户是谁，他们的主要工作流程是什么？
               - 预期的用户量和性能需求是什么？
               - 是否需要与现有系统集成？
               📝 简报: [生成结构化需求文档]"
```

### 示例2：认证系统改进
```
标准模式: "也许我们可以改进认证系统"
头脑风暴模式: "🔍 让我们一起来探索:
               - 用户面临的具体认证挑战是什么？
               - 当前与期望的用户体验对比？
               - 安全要求和合规需求是什么？
               - 时间线和资源限制？
               ✅ 成果: 清晰的改进路线图"
```

### 示例3：移动应用开发
```
标准模式: "需要一个移动应用"
头脑风暴模式: "📱 深入探讨:
               - 目标平台是iOS、Android还是跨平台？
               - 核心功能和次要功能分别是什么？
               - 离线功能是否必要？
               - 数据同步策略如何设计？
               📋 需求: [详细的功能需求规格]"
```

## 🎯 最佳实践

### 1. 提问技巧
- **开放式问题**: 避免是/否问题，鼓励详细回答
- **渐进式深入**: 从宏观到微观，逐步深入
- **场景化提问**: 基于具体使用场景提问
- **优先级探索**: 帮助用户确定需求优先级

### 2. 倾听策略
- **主动倾听**: 真正理解用户的意图和需求
- **确认理解**: 定期确认对需求的理解是否正确
- **记录要点**: 及时记录关键信息和决策点
- **总结反馈**: 定期总结讨论成果

### 3. 需求管理
- **分类整理**: 将需求按类别进行整理
- **优先级排序**: 帮助用户确定需求优先级
- **可行性评估**: 评估各项需求的可行性
- **冲突解决**: 帮助解决需求间的冲突
