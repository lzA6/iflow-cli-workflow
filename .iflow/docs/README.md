# iFlow 终极工作流

欢迎来到 iFlow 终极工作流项目。本项目的目标是重构和统一现有的 iFlow 工作流，创建一个单一、高效、自进化的“终极万能专家”系统。

## 核心理念

本工作流基于 T-MIA（超凡元智能代理系统）架构，其核心原则包括：

*   **内核化 (Kernelization):** 核心功能被封装为独立的、可优化的内核。
*   **计算验证 (Computational Validation):** 所有关键决策都通过形式化验证和基准测试。
*   **合规强制 (Compliance Enforcement):** ARCK 强制执行预设的指导原则和安全规范。
*   **递归学习 (Recursive Learning):** 系统能够从经验中学习，实现智能的指数级放大。

## 架构

新的工作流架构围绕以下核心组件构建：

*   `iflow/core`: 包含 T-MIA 的四个核心内核：
    *   `dkcm_system.py`: 动态知识与上下文管理 (DKCM)
    *   `arq_engine.py`: 专注推理与合规内核 (ARCK)
    *   `male_system.py`: 多代理协同与元学习引擎 (MALE)
    *   `rpfv_system.py`: 鲁棒性、性能与形式化验证 (RPFV)
*   `iflow/agents`: 包含“终极万能专家”智能体以及未来可能扩展的专业智能体。
*   `iflow/tools`: 统一的、经过重构的工具集。
*   `iflow/workflows`: 与新架构对齐的、精炼的工作流定义。
*   `iflow/config`: 全局和工作流的配置文件。
*   `iflow/docs`: 项目文档。
*   `iflow/tests`: 自动化测试框架。

## 目标

*   **消除冗余:** 清理所有过时和重复的文件，保持项目整洁。
*   **统一架构:** 将所有功能整合到 T-MIA 核心架构下。
*   **提升效率:** 通过智能体融合和工作流优化，大幅提升性能。
*   **自我进化:** 使系统能够从过去的交互中学习和改进。
