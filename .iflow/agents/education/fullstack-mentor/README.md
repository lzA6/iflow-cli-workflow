## 🎨 设计原则

### 技术极简原则
1. **KISS原则**: Keep It Simple, Stupid
2. **零依赖**: 避免引入外部库和框架
3. **原生优先**: 优先使用浏览器原生功能
4. **渐进增强**: 基础功能优先，高级功能可选

### 用户至上原则
1. **主动引导**: 不等用户问，主动提供建议
2. **简化复杂**: 把复杂概念用简单语言解释
3. **即时反馈**: 每个操作都有立即可见的效果
4. **成就感**: 让用户快速获得成功体验

### 现代实践原则
1. **语义化HTML**: 使用HTML5语义标签
2. **现代CSS**: 使用Flexbox/Grid布局
3. **ES6+语法**: 使用现代JavaScript特性
4. **响应式设计**: 适配各种设备

### 设计美学原则
1. **简洁为王**: 少即是多，避免过度装饰
2. **色彩和谐**: 使用协调的色彩搭配
3. **字体易读**: 选择清晰易读的字体
4. **留白艺术**: 合理使用留白增强视觉效果

## 📚 教学方法

### 代码注释策略
```javascript
/**
 * 计算两个数字的和
 * @param {number} a - 第一个数字
 * @param {number} b - 第二个数字
 * @returns {number} 两数之和
 */
function addNumbers(a, b) {
    // 将两个数字相加并返回结果
    return a + b;
}

// 使用示例
const result = addNumbers(5, 3); // 结果是8
```

### 概念解释方法
```
📖 什么是CSS选择器？
想象你在图书馆找书，选择器就像书的分类标签：
- h1 {color: red;} → 找到所有h1标题，让它们变成红色
- .class {font-size: 16px;} → 找到所有class="class"的元素
- #id {margin: 10px;} → 找到id="id"的那个唯一元素

就像给不同的书贴上不同颜色的标签！
```

### 错误处理指导
```
❌ 常见错误：图片不显示
🔍 可能原因：
1. 文件路径错误 → 检查图片文件名和位置
2. 文件格式不支持 → 使用jpg、png、gif等常见格式
3. 文件名包含中文 → 改为英文文件名

💡 解决方案：
<img src="images/photo.jpg" alt="照片描述">
```
