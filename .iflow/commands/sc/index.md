# /sc:index - 项目文档

## 触发条件
- 项目文档创建和维护要求
- 知识库生成和组织需求
- API文档和结构分析要求
- 交叉引用和导航增强请求

## 使用方法
```
/sc:index [目标] [--type docs|api|structure|readme] [--format md|json|yaml]
```

## 行为流程
1. **分析**: 检查项目结构并识别关键文档组件
2. **组织**: 应用智能组织模式和交叉引用策略
3. **生成**: 使用框架特定模式创建全面的文档
4. **验证**: 确保文档完整性和质量标准
5. **维护**: 更新现有文档，同时保留手动添加和自定义

关键行为：
- 基于文档范围和复杂性的多角色协调（架构师、抄写员、质量）
- Sequential MCP集成，用于系统化分析和全面的文档工作流
- Context7 MCP集成，用于框架特定模式和文档标准
- 具有交叉引用功能和自动化维护的智能组织

## MCP集成
- **Sequential MCP**: 复杂多步项目分析和系统化文档生成
- **Context7 MCP**: 框架特定文档模式和既定标准
- **角色协调**: 架构师（结构）、抄写员（内容）、质量（验证）

## 工具协调
- **Read/Grep/Glob**: 项目结构分析和内容提取，用于文档生成
- **Write**: 具有智能组织和交叉引用的文档创建
- **TodoWrite**: 复杂多组件文档工作流的进度跟踪
- **Task**: 需要系统化协调的大规模文档的委托

## 关键模式
- **结构分析**: 项目检查 → 组件识别 → 逻辑组织 → 交叉引用
- **文档类型**: API文档 → 结构文档 → README → 知识库方法
- **质量验证**: 完整性评估 → 准确性验证 → 标准合规性 → 维护计划
- **框架集成**: Context7模式 → 官方标准 → 最佳实践 → 一致性验证

## 示例

### 项目结构文档
```
/sc:index project-root --type structure --format md
# 具有智能组织的全面项目结构文档
# 创建具有交叉引用和组件关系的导航结构
```

### API文档生成
```
/sc:index src/api --type api --format json
# 具有系统化分析和验证的API文档
# 抄写员和质量角色确保完整性和准确性
```

### 知识库创建
```
/sc:index . --type docs
# 具有项目特定模式的交互式知识库生成
# 架构师角色提供结构组织和交叉引用
```

## 边界限制

**将会执行:**
- 生成具有智能组织和交叉引用的全面项目文档
- 应用多角色协调，进行系统化分析和质量验证
- 提供框架特定模式和既定文档标准

**不会执行:**
- 未经明确更新许可覆盖现有手动文档
- 在没有适当项目结构分析和验证的情况下生成文档
- 绕过既定的文档标准或质量要求
