# /sc:troubleshoot - 问题诊断和解决

## 触发条件
- 代码缺陷和运行时错误调查请求
- 构建失败分析和解决需求
- 性能问题诊断和优化要求
- 部署问题分析和系统行为调试

## 使用方法
```
/sc:troubleshoot [问题] [--type bug|build|performance|deployment] [--trace] [--fix]
```

## 行为流程
1. **分析**: 检查问题描述并收集相关的系统状态信息
2. **调查**: 通过系统模式分析识别潜在的根本原因
3. **调试**: 执行结构化调试程序，包括日志和状态检查
4. **提出**: 通过影响评估和风险评估验证解决方案方法
5. **解决**: 应用适当的修复并验证解决效果

关键行为：
- 具有假设检验和证据收集的系统根本原因分析
- 多领域故障排除（代码、构建、性能、部署）
- 具有全面问题分析的结构化调试方法
- 具有验证和文档的安全修复应用

## 工具协调
- **Read**: 日志分析和系统状态检查
- **Bash**: 诊断命令执行和系统调查
- **Grep**: 错误模式检测和日志分析
- **Write**: 诊断报告和解决方案文档

## 关键模式
- **Bug调查**: 错误分析 → 堆栈跟踪检查 → 代码检查 → 修复验证
- **构建故障排除**: 构建日志分析 → 依赖检查 → 配置验证
- **性能诊断**: 指标分析 → 瓶颈识别 → 优化建议
- **部署问题**: 环境分析 → 配置验证 → 服务验证

## 示例

### 代码Bug调查
```
/sc:troubleshoot "用户服务中的空指针异常" --type bug --trace
# 错误上下文和堆栈跟踪的系统分析
# 识别根本原因并提供有针对性的修复建议
```

### 构建失败分析
```
/sc:troubleshoot "TypeScript编译错误" --type build --fix
# 分析构建日志和TypeScript配置
# 自动应用常见编译问题的安全修复
```

### 性能问题诊断
```
/sc:troubleshoot "API响应时间下降" --type performance
# 性能指标分析和瓶颈识别
# 提供优化建议和监控指导
```

### 部署问题解决
```
/sc:troubleshoot "生产环境中的服务未启动" --type deployment --trace
# 环境和配置分析
# 部署要求和依赖项的系统验证
```

## 边界限制

**将会执行:**
- 使用结构化调试方法执行系统问题诊断
- 提供具有全面问题分析的验证解决方案方法
- 应用具有验证和详细解决方案文档的安全修复

**不会执行:**
- 在没有适当分析和用户确认的情况下应用有风险的修复
- 未经明确许可和安全验证修改生产系统
- 在不了解完整系统影响的情况下进行架构更改
