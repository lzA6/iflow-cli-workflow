# /sc:document - 专注文档生成

## 触发条件
- 特定组件、函数或功能的文档请求
- API文档和参考资料生成需求
- 代码注释和内联文档要求
- 用户指南和技术文档创建请求

## 使用方法
```
/sc:document [目标] [--type inline|external|api|guide] [--style brief|detailed]
```

## 行为流程
1. **分析**: 检查目标组件结构、接口和功能
2. **识别**: 确定文档要求和目标受众上下文
3. **生成**: 根据类型和样式创建适当的文档内容
4. **格式化**: 应用一致的结构和组织模式
5. **集成**: 确保与现有项目文档生态系统的兼容性

关键行为：
- 代码结构分析，包括API提取和用法模式识别
- 多格式文档生成（内联、外部、API参考、指南）
- 一致的格式化和交叉引用集成
- 语言特定的文档模式和约定

## 工具协调
- **Read**: 组件分析和现有文档审查
- **Grep**: 引用提取和模式识别
- **Write**: 具有正确格式的文档文件创建
- **Glob**: 多文件文档项目和组织

## 关键模式
- **内联文档**: 代码分析 → JSDoc/docstring生成 → 内联注释
- **API文档**: 接口提取 → 参考资料 → 用法示例
- **用户指南**: 功能分析 → 教程内容 → 实现指导
- **外部文档**: 组件概述 → 详细规范 → 集成说明

## 示例

### 内联代码文档
```
/sc:document src/auth/login.js --type inline
# 生成带有参数和返回描述的JSDoc注释
# 为函数和类添加全面的内联文档
```

### API参考生成
```
/sc:document src/api --type api --style detailed
# 创建包含端点和模式的全面API文档
# 生成用法示例和集成指南
```

### 用户指南创建
```
/sc:document payment-module --type guide --style brief
# 创建以用户为中心的文档，包含实用示例
# 专注于实现模式和常见用例
```

### 组件文档
```
/sc:document components/ --type external
# 为组件库生成外部文档文件
# 包括属性、用法示例和集成模式
```

## 边界限制

**将会执行:**
- 为特定组件和功能生成专注的文档
- 根据目标受众需求创建多种文档格式
- 与现有文档生态系统集成并保持一致性

**不会执行:**
- 在没有适当代码分析和上下文理解的情况下生成文档
- 覆盖现有文档标准或项目特定约定
- 创建暴露敏感实现细节的文档
