# /sc:analyze - 代码分析与质量评估

## 触发条件
- 项目或特定组件的代码质量评估请求
- 安全漏洞扫描和合规验证需求
- 性能瓶颈识别和优化规划
- 架构审查和技术债务评估需求

## 使用方法
```
/sc:analyze [目标] [--focus quality|security|performance|architecture] [--depth quick|deep] [--format text|json|report]
```

## 行为流程
1. **发现**: 使用语言检测和项目分析对源文件进行分类
2. **扫描**: 应用特定领域分析技术和模式匹配
3. **评估**: 生成具有严重性评级和影响评估的优先级发现
4. **推荐**: 创建具有实施指导的可操作建议
5. **报告**: 提供包含指标和改进路线图的综合分析

关键行为：
- 结合静态分析和启发式评估的多领域分析
- 智能文件发现和特定语言模式识别
- 基于严重性的发现和建议优先级排序
- 包含指标、趋势和可操作见解的综合报告

## 工具协调
- **Glob**: 文件发现和项目结构分析
- **Grep**: 模式分析和代码搜索操作
- **Read**: 源代码检查和配置分析
- **Bash**: 外部分析工具执行和验证
- **Write**: 报告生成和指标文档化

## 关键模式
- **领域分析**: 质量/安全/性能/架构 → 专业评估
- **模式识别**: 语言检测 → 适当的分析技术
- **严重性评估**: 问题分类 → 优先级建议
- **报告生成**: 分析结果 → 结构化文档

## 示例

### 综合项目分析
```
/sc:analyze
# 整个项目的多领域分析
# 生成包含关键发现和路线图的综合报告
```

### 专注安全评估
```
/sc:analyze src/auth --focus security --depth deep
# 认证组件的深度安全分析
# 包含详细修复指导的漏洞评估
```

### 性能优化分析
```
/sc:analyze --focus performance --format report
# 性能瓶颈识别
# 生成包含优化建议的HTML报告
```

### 快速质量检查
```
/sc:analyze src/components --focus quality --depth quick
# 组件目录的快速质量评估
# 识别代码异味和可维护性问题
```

## 边界限制

**将会执行:**
- 跨多个领域执行综合静态代码分析
- 生成具有可操作建议的严重性评级发现
- 提供包含指标和改进指导的详细报告

**不会执行:**
- 执行需要代码编译或运行时的动态分析
- 未经明确用户同意修改源代码或应用修复
- 超出导入和使用模式分析外部依赖
