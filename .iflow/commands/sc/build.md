# /sc:build - 项目构建和打包

## 触发条件
- 不同环境的项目编译和打包请求
- 构建优化和构件生成需求
- 构建过程中的错误调试
- 部署准备和构件打包需求

## 使用方法
```
/sc:build [目标] [--type dev|prod|test] [--clean] [--optimize] [--verbose]
```

## 行为流程
1. **分析**: 项目结构、构建配置和依赖清单
2. **验证**: 构建环境、依赖项和必需工具链组件
3. **执行**: 具有实时监控和错误检测的构建过程
4. **优化**: 构建构件、应用优化和最小化包大小
5. **打包**: 生成部署构件和全面构建报告

关键行为：
- 配置驱动的构建编排，具有依赖验证
- 智能错误分析，具有可操作的解决指导
- 环境特定优化（dev/prod/test配置）
- 全面构建报告，具有时间指标和构件分析

## MCP集成
- **Playwright MCP**: 构建期间自动激活用于构建验证和UI测试
- **DevOps工程师角色**: 激活用于构建优化和部署准备
- **增强功能**: 构建流水线集成、性能监控、构件验证

## 工具协调
- **Bash**: 构建系统执行和进程管理
- **Read**: 配置分析和清单检查
- **Grep**: 错误解析和构建日志分析
- **Glob**: 构件发现和验证
- **Write**: 构建报告和部署文档

## 关键模式
- **环境构建**: dev/prod/test → 适当的配置和优化
- **错误分析**: 构建失败 → 诊断分析和解决指导
- **优化**: 构件分析 → 大小减少和性能改进
- **验证**: 构建验证 → 质量门控和部署就绪性

## 示例

### 标准项目构建
```
/sc:build
# 使用默认配置构建整个项目
# 生成构件和全面构建报告
```

### 生产优化构建
```
/sc:build --type prod --clean --optimize
# 具有高级优化的清洁生产构建
# 压缩、树摇和部署准备
```

### 目标组件构建
```
/sc:build frontend --verbose
# 构建特定项目组件，具有详细输出
# 实时进度监控和诊断信息
```

### 开发构建与验证
```
/sc:build --type dev --validate
# 具有Playwright验证的开发构建
# UI测试和构建验证集成
```

## 边界限制

**将会执行:**
- 使用现有配置执行项目构建系统
- 提供全面错误分析和优化建议
- 生成具有详细报告的部署就绪构件

**不会执行:**
- 修改构建系统配置或创建新构建脚本
- 安装缺失的构建依赖或开发工具
- 执行超出构件准备的部署操作
