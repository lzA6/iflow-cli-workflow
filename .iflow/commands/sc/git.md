# /sc:git - Git操作

## 触发条件
- Git仓库操作：状态、添加、提交、推送、拉取、分支
- 需要智能提交消息生成
- 仓库工作流优化请求
- 分支管理和合并操作

## 使用方法
```
/sc:git [操作] [参数] [--smart-commit] [--interactive]
```

## 行为流程
1. **分析**: 检查仓库状态和工作目录更改
2. **验证**: 确保操作适用于当前Git上下文
3. **执行**: 使用智能自动化运行Git命令
4. **优化**: 应用智能提交消息和工作流模式
5. **报告**: 提供状态和下一步指导

关键行为：
- 根据更改分析生成规范化提交消息
- 应用一致的分支命名约定
- 通过引导式解决方案处理合并冲突
- 提供清晰的状态摘要和工作流建议

## 工具协调
- **Bash**: Git命令执行和仓库操作
- **Read**: 仓库状态分析和配置审查
- **Grep**: 日志解析和状态分析
- **Write**: 提交消息生成和文档

## 关键模式
- **智能提交**: 分析更改 → 生成规范化提交消息
- **状态分析**: 仓库状态 → 可操作建议
- **分支策略**: 一致的命名和工作流强制执行
- **错误恢复**: 冲突解决和状态恢复指导

## 示例

### 智能状态分析
```
/sc:git status
# 分析仓库状态并提供更改摘要
# 提供下一步和工作流建议
```

### 智能提交
```
/sc:git commit --smart-commit
# 根据更改分析生成规范化提交消息
# 应用最佳实践和一致的格式
```

### 交互式操作
```
/sc:git merge feature-branch --interactive
# 引导式合并，提供冲突解决帮助
```

## 边界限制

**将会执行:**
- 使用智能自动化执行Git操作
- 根据更改分析生成规范化提交消息
- 提供工作流优化和最佳实践指导

**不会执行:**
- 未经明确授权修改仓库配置
- 未经确认执行破坏性操作
- 处理需要手动干预的复杂合并
