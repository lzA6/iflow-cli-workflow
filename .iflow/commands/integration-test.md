# 系统集成测试工作流

## 功能概述

这个工作流执行全面的系统集成测试，包括端到端流程测试、模块间接口测试、数据流验证、系统兼容性测试等关键集成指标的验证。

## 执行步骤

### 1. 集成测试环境初始化
- 激活超级思考模式
- 初始化集成测试环境
- 加载测试配置和数据

### 2. 全面集成测试
- **端到端流程测试**: 完整业务流程验证
- **模块间接口测试**: API接口和数据交换测试
- **数据流验证**: 数据传递和转换验证
- **系统兼容性测试**: 跨系统集成测试

### 3. 集成问题分析和优化
- 识别集成问题
- 提供解决方案
- 生成测试报告
- 制定改进计划

## 输入参数

```
/integration-test [可选参数]
```

**可选参数**:
- `--end-to-end`: 端到端流程测试
- `--interface`: 接口集成测试
- `--data-flow`: 数据流验证
- `--compatibility`: 兼容性测试
- `--regression`: 回归测试
- `--load`: 负载集成测试

## 集成测试标准

### 优秀 (Excellent)
- 集成评分 ≥ 95分
- 所有关键接口正常
- 数据流完全一致
- 无集成故障

### 良好 (Good)
- 集成评分 ≥ 85分
- 主要接口正常
- 数据流基本一致
- 轻微集成问题

### 一般 (Fair)
- 集成评分 ≥ 70分
- 部分接口存在问题
- 数据流存在差异
- 有集成故障但可修复

### 需改进 (Poor)
- 集成评分 < 70分
- 关键接口故障
- 数据流严重不一致
- 重大集成问题

## 注意事项

1. 集成测试可能需要15-30分钟
2. 端到端测试需要完整的测试数据
3. 某些测试可能影响生产环境，请谨慎执行
4. 发现的集成问题将被详细记录和跟踪
